Make Your First API Call

Choose your language:

cURL 
curl -X POST https://api.realfact.tech/v1/summarize \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Your document text here...",
    "max_length": 100
  }'
Python 
import requests

url = "https://api.realfact.tech/v1/summarize"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "text": "Your document text here...",
    "max_length": 100
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
JavaScript (Node.js) 
const response = await fetch("https://api.realfact.tech/v1/summarize", {
  method: "POST",
  headers: {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    text: "Your document text here...",
    max_length: 100,
  }),
});

const data = await response.json();
console.log(data);
Java (HttpClient) 
var client = HttpClient.newHttpClient();
var body = """
  {"text": "Your document text here...", "max_length": 100}
  """;
var request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.realfact.tech/v1/summarize"))
    .header("Authorization", "Bearer YOUR_API_KEY")
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(body))
    .build();

var response = client.send(request,
    HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());

Error Code Reference

All API errors return a JSON body with error.code, error.message, and HTTP status code.

Error Response Format

Error Response Example
{
  "error": {
    "code": "invalid_api_key",
    "message": "The API key provided is invalid.",
    "status": 401
  }
}

Authentication Errors (4xx)

Status Code Description How to Fix
401 invalid_api_key API key is invalid or missing Check your API key. Make sure to include Bearer prefix.
401 key_revoked API key has been revoked Re-activate your app or create a new one.
403 sandbox_restricted Sandbox key used on production endpoint Submit your app for production approval.
403 app_not_approved App is pending approval or rejected Wait for approval or resubmit if rejected.

Rate Limit Errors

Status Code Description How to Fix
429 rate_limit_exceeded Too many requests in the time window Wait and retry with exponential backoff. Check Retry-After header.
429 sandbox_quota_exceeded Sandbox daily limit (30 req/day) reached Wait until tomorrow or submit for production.

Request Errors

Status Code Description How to Fix
400 invalid_request Request body is malformed or missing required fields Check the request body against the API docs.
400 payload_too_large Request exceeds max payload size Sandbox: 1 MB max. Production: 10 MB max.
404 endpoint_not_found API endpoint does not exist Check URL path and HTTP method.

Server Errors

Status Code Description How to Fix
500 internal_error Unexpected server error Retry after a few seconds. If persistent, check status page.
503 service_unavailable Service temporarily unavailable Retry with exponential backoff. Check status page.
504 gateway_timeout Request timed out (processing took too long) Reduce input size or use batch processing.
